🎯 WHAT IS FOODFLOW?

FoodFlow is a food donation platform that connects:
    • DONORS (people with extra food)
    • NGOs (organizations that need food)
    • DRIVERS (people who deliver food)
    • ADMINS (platform managers)

🔄 HOW THEY WORK TOGETHER

┌─────────────────────────────────────────────────────────────────────────────┐
│ SCENARIO: User Signs Up                                                     │
└─────────────────────────────────────────────────────────────────────────────┘

1. USER OPENS BROWSER
   └─ Goes to http://localhost:3000/signup
   └─ Sees signup form (from FRONTEND)

2. USER FILLS FORM
   └─ Email: user@example.com
   └─ Password: SecurePass123!
   └─ Name: John Doe
   └─ Role: DONOR

3. USER CLICKS "SIGN UP"
   └─ FRONTEND receives click event
   └─ FRONTEND validates input
   └─ FRONTEND hashes password
   └─ FRONTEND creates user in DATABASE
   └─ FRONTEND generates verification token
   └─ FRONTEND stores token in DATABASE

4. FRONTEND CALLS BACKEND
   └─ Sends HTTP request to http://localhost:8000/api/send-verification-email
   └─ Includes: email, name, verification URL

5. BACKEND RECEIVES REQUEST
   └─ BACKEND builds HTML email
   └─ BACKEND connects to Brevo SMTP
   └─ BACKEND sends email

6. BREVO SENDS EMAIL
   └─ Email delivered to user@example.com
   └─ User receives in inbox

7. USER CLICKS EMAIL LINK
   └─ Link: http://localhost:3000/auth/verify-email?token=abc123&email=user@example.com
   └─ FRONTEND receives request
   └─ FRONTEND calls verification API

8. FRONTEND VERIFIES TOKEN
   └─ Checks token in DATABASE
   └─ Checks token not expired
   └─ Marks email as verified in DATABASE
   └─ Deletes token from DATABASE

9. USER SEES SUCCESS
   └─ "Email verified! Redirecting to login..."
   └─ User can now login

═══════════════════════════════════════════════════════════════════════════════

📊 COMMUNICATION BETWEEN FRONTEND AND BACKEND

Frontend → Backend (HTTP Requests)
    POST http://localhost:8000/api/send-verification-email
    {
      "email": "user@example.com",
      "name": "John Doe",
      "verification_url": "http://localhost:3000/auth/verify-email?token=abc123"
    }

Backend → Frontend (HTTP Response)
    HTTP 200 OK
    {
      "success": true,
      "message": "Email sent successfully"
    }

Both ← Database (SQL Queries)
    Frontend: SELECT * FROM users WHERE email = 'user@example.com'
    Backend: SELECT * FROM verification_tokens WHERE token = 'abc123'

🚀 STARTING THE APPLICATION

STEP 1: Start Backend
    Terminal 1:
    cd backend
    python -m uvicorn main:app --reload --port 8000
    
    What happens:
    • Python starts FastAPI server
    • Connects to PostgreSQL database
    • Loads all API routes
    • Starts email queue worker
    • Starts background scheduler
    
    Output:
    ✅ Database connected
    ✅ Email worker started
    ✅ Background scheduler started
    INFO: Application startup complete.

STEP 2: Start Frontend
    Terminal 2:
    cd frontend
    npm run dev
    
    What happens:
    • Node.js starts Next.js development server
    • Compiles React components
    • Sets up hot reload
    • Initializes authentication
    
    Output:
    ▲ Next.js 14.0.0
    - Local: http://localhost:3000

STEP 3: Open Browser
    Go to: http://localhost:3000
    
    What happens:
    • Browser loads frontend
    • Frontend loads React components
    • Frontend initializes authentication
    • User sees signup page

🔌 PORTS AND CONNECTIONS

Frontend Server
    ├─ Port: 3000
    ├─ URL: http://localhost:3000
    ├─ What: Web pages, user interface
    └─ Connects to: Backend (port 8000), Database

Backend Server
    ├─ Port: 8000
    ├─ URL: http://localhost:8000
    ├─ What: API endpoints, email service
    └─ Connects to: Database, Brevo SMTP, Redis

Database
    ├─ Port: 5432 (default PostgreSQL)
    ├─ Host: localhost
    ├─ What: Data storage
    └─ Used by: Frontend and Backend

Brevo SMTP
    ├─ Host: smtp-relay.brevo.com
    ├─ Port: 587
    ├─ What: Email delivery service
    └─ Used by: Backend

📱 USER JOURNEY

1. DONOR SIGNS UP
   └─ Frontend: Signup form
   └─ Backend: Sends verification email
   └─ Database: Stores user and token
   └─ Email: Verification link sent

2. DONOR VERIFIES EMAIL
   └─ Frontend: Verification page
   └─ Database: Marks email as verified
   └─ Frontend: Redirects to login

3. DONOR LOGS IN
   └─ Frontend: Login form
   └─ Database: Verifies credentials
   └─ Frontend: Creates session
   └─ Frontend: Redirects to dashboard

4. DONOR VIEWS DASHBOARD
   └─ Frontend: Fetches analytics
   └─ Database: Returns dashboard data
   └─ Frontend: Displays dashboard

5. DONOR CREATES LISTING
   └─ Frontend: Listing form
   └─ Database: Stores listing
   └─ Frontend: Shows confirmation

6. NGO CLAIMS LISTING
   └─ Frontend: NGO sees listing
   └─ Database: Creates claim
   └─ Frontend: Shows claim status

7. DRIVER ASSIGNED
   └─ Backend: Assigns driver
   └─ Database: Creates delivery
   └─ Frontend: Shows delivery

8. DRIVER UPDATES LOCATION
   └─ Frontend: Driver tracking page
   └─ Backend: Receives location
   └─ Database: Stores location
   └─ Frontend: Updates map in real-time

9. DONOR TRACKS DELIVERY
   └─ Frontend: Tracking page
   └─ Database: Fetches delivery data
   └─ Frontend: Shows map with driver location

10. DELIVERY COMPLETED
    └─ Backend: Marks as delivered
    └─ Database: Updates status
    └─ Frontend: Shows completion

🔐 DATA FLOW SECURITY

User Input
    ↓
Frontend Validation
    ├─ Check required fields
    ├─ Validate email format
    ├─ Check password strength
    ↓
Frontend Processing
    ├─ Hash password (bcrypt)
    ├─ Sanitize input
    ↓
API Request
    ├─ HTTPS (in production)
    ├─ JWT token in header
    ↓
Backend Validation
    ├─ Verify JWT token
    ├─ Validate input again
    ├─ Check permissions
    ↓
Database Query
    ├─ Parameterized queries (Prisma)
    ├─ Prevent SQL injection
    ↓
Database Storage
    ├─ Encrypted passwords
    ├─ Secure tokens
    ↓
Response
    ├─ No sensitive data in response
    ├─ Error messages don't leak info

🎓 KEY CONCEPTS

API (Application Programming Interface)
    └─ Way for frontend to talk to backend
    └─ Frontend sends HTTP requests
    └─ Backend sends HTTP responses

Database
    └─ Where all data is stored
    └─ Both frontend and backend use same database
    └─ Data persists even after restart

Authentication
    └─ Verifying user identity
    └─ JWT tokens store user info
    └─ Secure cookies prevent unauthorized access

Email Verification
    └─ Confirms user owns email address
    └─ One-time use token
    └─ Expires after 24 hours

Real-time Tracking
    └─ Live location updates
    └─ WebSocket or polling
    └─ Map updates every 2 seconds
